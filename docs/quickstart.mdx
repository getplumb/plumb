# Quickstart

Get Plumb memory working with your AI assistant in under 5 minutes.

## For OpenClaw Users (Recommended)

If you use OpenClaw, install the plugin for zero-config memory integration:

```bash
openclaw plugin install @getplumb/openclaw-plugin
```

That's it. Memory gets auto-injected into every conversation. Skip to [Verify](#verify) to confirm it's working.

See [OpenClaw Plugin](/openclaw-plugin) for advanced configuration options.

## For MCP-Compatible Clients

If you use Claude Desktop, Claude Code, Cursor, or another MCP client, install the MCP server:

### 1. Install the MCP Server

```bash
npm install -g @getplumb/mcp-server
```

This installs `plumb-mcp` globally, making it available to any MCP client.

### 2. Configure Your LLM Provider

Plumb needs an LLM for fact extraction. Run the setup wizard:

```bash
plumb setup
```

Follow the prompts to configure OpenAI, Anthropic, Ollama, or an OpenAI-compatible provider. This writes environment variables to your shell RC file.

### 3. Add to MCP Client Config

Connect Plumb to your tool:

**Claude Desktop** — Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "plumb": {
      "command": "plumb-mcp",
      "args": []
    }
  }
}
```

Config location:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

**Claude Code** — Run:

```bash
plumb connect claude-code
```

This auto-registers Plumb via `claude mcp add` and updates `~/.claude/CLAUDE.md` with usage instructions.

**Cursor** — Add to settings.json:

```json
{
  "mcp.servers": {
    "plumb": {
      "command": "plumb-mcp",
      "args": []
    }
  }
}
```

**Other MCP clients** — Use:

```bash
plumb connect <tool>
```

Supported: `claude-desktop`, `claude-code`, `cursor`, `openclaw`.

### 4. Restart Your Client

Restart Claude Desktop, Claude Code, or Cursor to load the MCP server.

## Verify

Check that Plumb is working:

### For OpenClaw Users

```bash
plumb status
```

You should see memory statistics (fact count, raw log count, storage size).

### For MCP Client Users

In your AI conversation, the following tools should appear:
- `memory_search` — Search stored facts and conversations
- `memory_store` — Explicitly store a fact
- `memory_delete` — Remove a fact by ID
- `memory_status` — Get memory statistics

Ask the AI: *"What memory tools do you have?"* or *"Call memory_status"* to confirm.

## First Memory Stored

Start a conversation with your AI:

> You: "I prefer using pnpm for Node.js projects"
>
> AI: "Got it, I'll remember that you prefer pnpm."

Behind the scenes:
1. Plumb stores the exchange in Layer 1 (raw log)
2. Fact extraction runs async, creating a fact: `user prefers_package_manager pnpm` (confidence: 0.9)
3. Next conversation, the AI retrieves this fact automatically

Output:

```
  user                 (1 fact)

```

## Export Your Memory

Export all data to JSON + Markdown:

```bash
plumb export
```

Creates `plumb-export-<timestamp>/` with:
- `facts.json` — All extracted facts
- `facts.md` — Human-readable fact list
- `raw-log.json` — Full conversation log
- `raw-log.md` — Human-readable log
